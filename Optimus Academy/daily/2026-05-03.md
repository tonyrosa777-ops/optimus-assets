### Detailed Notes

#### April 2026 maturity step — what actually shipped
- Multi-agent task orchestration: a claw can now call multiple agents over the course of a single task "really seamlessly" — i.e., a Manager agent delegates to specialist agents and assembles the result.
- Task organization and reporting: the claw maturely tracks task state and reports back rather than just running open-ended.
- Messaging-channel awareness: the claw knows how to respond correctly inside a Slack thread (and presumably the other supported channels — WhatsApp, Telegram, Discord, Signal, iMessage, Teams per the wider OpenClaw ecosystem).
- Nate's metaphor: "OpenClaw was a teenager who got the keys to the car in February and went nuts. The teenager grew up, got a prefrontal cortex, and is now a responsible citizen. Your claw can do serious work."

#### Underlying architecture (filled in via enrichment, not stated in source)
- Built by Peter Steinberger (PSPDFKit founder); ~310k GitHub stars, 58k forks, 1,200+ contributors as of April 2026.
- Hierarchical agent topology: Manager → Researcher → Writer → Editor pattern. Each agent runs in an isolated Docker container; agents communicate via well-defined RPC interfaces. This is what makes the multi-agent orchestration claim concrete — it's not a single LLM with tool use, it's distinct containerized agents.
- Node-graph, event-driven, local-first design. Manifest-driven security with eBPF kernel enforcement.
- Multi-model abstraction supports Claude Opus 4.7 (200K context), Google Gemini, local LLMs and Qwen via Ollama. Model selection is configurable per-agent.
- Connects to WhatsApp, Telegram, Slack, Discord, Signal, iMessage, Teams as messaging substrates — not "another chat UI," but a layer that plugs into the channels users already live in.

#### Builder thesis — multiple models per claw, not one
- "We should have a lot of models for claw" — Nate's central point.
- Cites Google Gemma as a strong local model option for the claw — not just frontier-API models.
- Frames the labs as "setting the agenda" in ways that distract builders. OpenAI is being generous with terms right now, Anthropic is not — but assume that see-saws.
- Action: build a claw that is YOURS, regardless of what model makers do. Model abstraction at the orchestration layer is the moat.

#### Companion project — Open Brain (OB1)
- Nate is shipping expansions to his Open Brain repo (github.com/NateBJones-Projects/OB1) to make multi-model claws easier to run.
- OB1 is a self-hosted shared persistent memory database for multiple AIs (Claude, ChatGPT, Cursor, etc.). Uses Supabase + OpenRouter; runs at $0.10–0.30/month on free-tier cloud.
- Distinct from OpenClaw — OB1 is the memory layer, OpenClaw is the orchestration layer. Both are self-hosted-first.

#### Ecosystem context (enrichment — for situational awareness)
- "OpenClaude" is a separate, looser term — informal community wrappers, not the same project as OpenClaw.
- Adjacent orchestration platforms: Ruflo (Claude-native swarm orchestration), OpenWork (Claude Cowork alternative), Paperclip (orchestrates OpenClaw + Claude Code into "company structure" with org charts/budgets/governance).
